Sharing Cleansing Knowledge

It's an open source data process framework under a micro-service architecture. The purpose of this application is to let people exchange the knowledge of hwo to clean the data and applying those knowledge to the data.

Algorithm

An algorithm is a data process logic.

Algorithm has its category(DataQuality, DataCleansing, DataMining, etc.) and the domain (Contract Information, Product Name, etc.)
Algorithm, which is embeded in an EngineAgent in DataMining Cloud. Will be published as docker image.

Model

A model is the setting or knowledge base of a algorithm.
Model has its type, locale and version. Model can be created based on an existed model or build from a Service or Flow.
Model and job can be shared and archive.
Model and job can be exported, in someway. A CVS file, a SQL script or a AWS S3 ticket, etc.

Job

A Process is a combination of algorithm and model.
A Job is a banch of processes with some flow control.
A Job could be deployied to the DMC Engine as a Flow.
Flow could last for a very long period(10h?).
If a flow's running is very quick(less then 10s), that job can be published to ServiceProxy as a REST Service. Every calling will trigger a new flow with the job.
